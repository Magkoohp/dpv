[v0.9] - 22 Oct 2022

This is the last point/minor version before v1 release. It makes the following changes:

  • Location: added City, PrivateLocation, PubliclyLocation
  • Legal Basis: added LegitimateInterestOfDataSubject
  • Organisational Measures: added ReviewImpactAssessmentAdequacy, ReviewProcedure, ReviewImpactAssessmentConformance, ConsultationWithDataSubjectRepresentative, ReviewImpactAssessment
  • Personal Data: renamed PseudoAnonymisedData to PseudonymisedData
  • Processing Context: added ScoringOfIndividuals and EvaluationOfIndividuals
  • Processing: renamed PseudoAnonymise to Pseudonymise
  • Purpose: renamed CreatePersonalizedRecommendations to CreatePersonalisedRecommendations
  • Status: added hasLawfulness, Lawfulness, NonConformant, LawfulnessUnkown, ConformanceStatus, Unlawful, Conformant, Lawful
  • Technical Measures: fixed pseudonymisation spelling in RNGPseudoanonymisation, MonotonicCounterPseudoanonymisation, EncryptionInRest, PseudoAnonymisation, Pseudoanonymisation. Added EncryptionInUse. Deleted duplicates of Pseudonymisation.
  • Rights: new module for Rights concepts, with additions of Active and Passive rights, and RightExercise information (work in progress).

[v0.8.2] - 06 Oct 2022

This is a minor release providing fixes to the release candidate. It makes the following changes.

  • Change concepts to use subtype instead of instance for the following: Context (Duration, Frequency), Entity (Organisation, DataSubject), Location.
  • This change also fixes issues where some OWL concepts were being declared as instances instead of subclasses.
  • Change Anonymisation terms for accuracy and unambiguity, where there is now no direct relation between anonymisation and pseudonymisation. The labels and parents have been updated to reflect this, and all terms have the common parent DataAnonymisationTechnique. CompleteAnonymisation has been deprecated as the concept is covered by Anonymisation. PseudoAnonymisation has been renamed to Pseudonymisation. The term DataRedaction has been added as a type of DataSanitisationTechnique.
  • Change term AutomatedProcessingWithHumanVerification to AutomatedProcessingWithHumanReview to reflect the broader use of review as compared to verify.
  • Split EvaluationScoring into distinct concepts representing Evaluation and Scoring (of individuals).
  • Fix label of Purpose ServiceUsageAnalytics - raised in #54 by @aucampia.
  • Provides better description for cases where a property does not have domain or range specified - raised in #52 by @bertvannuffelen.
  • Adds definition of Concept - raised in #51 by bertvannuffelen.
  • Change term AnonymisedDataWithinContext to better reflect its intended use as ContextuallyAnonymisedData.

[v0.8] - 26 Aug 2022

v0.8 makes several major changes to the DPV in preparation for progressing the work towards v1. It adds Consent states and types, and marks existing properties for deprecation in favour of new uniform properties that harmonise how notice and legal bases work in conjunction. The various statuses have now been organised in a dedicated module - Status. The technical and organisational measures are separated into dedicated modules, and contain substantial additions to each (148 total additions). The concepts related to various scales have been moved to the module ProcessingScale. This iteration adds the risk and rights extensions. It also expands the core risk vocabulary to express levels, severity, and likelihood.

v0.8 also changes how DPV serialisation files and documentation are generated. It significantly updates the templating system used internally, fixes several bugs and errors, and allows expressing arbitrary values for concepts - useful to express rdf:value and level, severity, likelihood associated with risk concepts. It also fixes issue #44 dpv-tech properties incorrect domain/range; and #37 #41 Adds property domains/ranges in HTML.

v0.8 introduces the generation of release zips that are intended to be uploaded to GitHub or other release platforms. These files are ignored by git.

[v0.2] - 14 Jan 2021

Release Notes

v0.2 represents an update to v0.1 in terms of added new concepts in the hierarchy, expanded base vocabulary, and some backwards-incompatible changes.

The core/base vocabulary has new concepts for Risk, Right, and DataSubjectRight. These are associated with PersonalDataHandling using the relations hasRisk and hasRight respectively.

Personal Data Categories has new concepts for BrowsingReferral, PaymentCard, PaymentCardNumber, PaymentCardExpiry, DeviceSoftware, DeviceOperatingSystem, DeviceApplications, AuthenticationHistory

Purposes has new concepts for Marketing, DirectMarketing, Advertising, PersonalisedAdvertising, UsageAnalytics, CommunicationForCustomerCare, LegalCompliance, Payment, SocialMediaMarketing, RegistrationAuthentication

Processing attributes, such as those for indicating systematic monitoring, large scale, etc. have been removed as boolean properties and added as classes. Full list: SystematicMonitoring, EvaluationScoring, MatchingCombining, AutomatedDecisionMaking, LargeScaleProcessing, InnovativeUseOfNewTechnologies. This is so as to enable assocaiting additional information and semantics for concepts, such as specific types, categories, relations, and enable broader use in context of processing. For example, indicating the specific algorithm or process for automated decision making - which could not be indicated using a boolean value. In addition to these, the concept for DataSource has been added. Generic properties added are: hasDataSource, hasAlgorithmicLogic, hasConsequences, hasHumanInvolvement.

Technical and Organisational Measures have new concepts for: SingleSignOn, ImpactAssessment, PIA, DPIA, Consultation, ConsultationWithAuthority. The concept RiskMitigationMeasure and its associated property mitigatesRisk has been added to indicate risk and mitigation. The existing properties for storage, location, duration, and measure implementation have been modified to avoid name duplication. For example, storage as a property and Storage as a concept are equivalent when used as HTML identifiers. To avoid this, the names have been prefixed, which also adds consistency to the vocabulary naming. The properties are: measureImplementedBy, hasStorage, hasLocation, hasDuration.

Entities added as new concepts include LegalEntity, DataSubProcessor, Child, DataProtectionOfficer, Representative, Authority, DataProtectionAuthority, VulnerableDataSubject. Generic properties for expressing information about entities added are: hasName, hasAddress, hasContact, hasIdentifier, hasRepresentative.

Release Notes

v0.1 consists of the first release in 18-JUN-2019, and includes subsequent updates till the last update on 19-NOV-2020. This version introduced the 'core' concepts of DPV, or the base vocabulary, and a taxonomical hierarchy of concepts expanding the vocabulary.

The various changes to the vocabulary have all been consolidated under a single version, namely v0.1. Moving forward, the vocabulary will utilise semantic versioning of the form MAJOR.MINOR.TRIVIAL. The v0.x notation denotes the ongoing progress towards a stable and mature vocabulary which will be finalised as v1.
